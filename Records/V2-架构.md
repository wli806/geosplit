## 0. 产品定位(MVP 范围)

输入奥克兰地址 → 输出:**GeoMap 多图层预览 + 3D 模型 + 分割收支平衡 + DEM/DSM/flood 成像**。
- **不做**自动房屋 layout(家/卧室/OLS/outlook 放置)→ 以后做**独立模块**。
- 产能只讲**理论**:推荐分割成几块、每块面积/层数/透水率/室外面积(启发式+经验公式),不摆房子。

## 1. 五层架构(单向依赖,自下而上)

```
L1 sources/   数据源:一源一文件,取数+缓存+解码+【溯源】,无业务逻辑
L2 domain/    领域计算:纯计算,无 HTTP 无渲染 —— 引擎价值核心
L3 render/    呈现:领域结果 → 图层负载;统一图层注册表住这里
L4 api/       FastAPI 瘦路由
L5 frontend/  Leaflet(2D)+ Cesium(3D),模式感知图层列表 + 2D→3D 转场
```

## 3. 三条铁律

1. **单向依赖**:L5→L4→L3→L2→L1,绝不反向。`domain/` 不许 import render/api/fastapi。
2. **domain 出抽象、render 出画法**:同一份领域结果,2D 一种画法、3D 一种画法,3D 引擎可换。
3. **数据带溯源**:凡 source 出来的字段都带 `Field{value, source, fetched_at, is_local}`。

## 4. 文件粒度规则

- **文件夹 = 层**;**文件 = 一个内聚职责**(一个数据源 / 一个领域能力),**不是一函数一文件**。
- 下划线前缀 `_xxx.py` = 该层私有工具,不对外。
- **跨层数据结构集中进 `contracts/`**(= 系统数据形状索引);模块内部私有类型就地定义。
- 准则:**高内聚(一起变的放一起)+ 低耦合(跨层只通过 contracts 对话)。**

## 5. 已锁定的技术决策

| 项 | 决定 |
|---|---|
| 2D 引擎 | **Leaflet**(切片地图观感好) |
| 3D 引擎 | **CesiumJS**(相机自由可钻地下看管网 + 原生地形/航拍/clampToGround;治好 deck.gl 卡视角) |
| 2D→3D 转场 | 对齐交接(Leaflet 视角→Cesium 俯视淡入)+ 镜头起身(flyTo 摇到斜俯视)+ 大楼生长(extrudedHeight 0→满) |
| 统一图层列表 | 一个左侧列表,随模式过滤;每层声明 `modes`;3D-only(HIRB 包络/地形面)在 2D 不出现 |
| CV/RV 源 | `sources/valuation.py` 主用官方 `Rates24_SnapshotReval`(取 LCV/LLV/LATESTVALUATIONDATE 最新一轮)→ 经验值兜底;**CV 当锚点不当卖价**;商用授权待向 council 确认一次 |
| 栅格库 | **numpy + tifffile + imagecodecs** 读 E 盘 DSM/DEM,避开 GDAL |
| 地形来源 | V2 用真 DSM/DEM 栅格(E:\Geomaps Data BASE);**不再**用 council Contours 反推 |
| 产能/收支 | `domain/subdivision.py`(N 扫描+每层可建面积+推荐)、`domain/economics.py`(break-even)两个独立模块 |
