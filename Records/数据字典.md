# 数据字典(场地输入模型)

> "场地输入模型"清单——所有可预见的输入因素都列在这里,即使 MVP 不用。
> 目的:让输入接口一开始就完整,以后填真值时**接口不变、公式不重写**。
> **对齐 V2 `contracts/`**:这里的字段最终落到 `contracts/site.py` 的 `ParcelData/ZoneData/ValuationData/SiteModel`,每个字段带溯源 `Field{value, source, fetched_at, is_local}`。
> 列说明:**MVP?** 本期是否实现;**默认值** 未实现时喂公式的中性值;**来源** 见《接口手册》。

## 图例
✅ MVP 实现 ｜ ⏳ 接口预留、MVP 给默认值 ｜ 🔮 以后扩充(含独立 layout 模块)

---

## 1. 地块 Parcel → `ParcelData`
| 字段 | MVP? | 默认值 | 来源 |
|---|---|---|---|
| 边界多边形 | ✅ | — | LINZ Primary Parcels |
| 净面积 net site area | ✅ | — | LINZ survey_area(扣 ROW/access) |
| 临街宽度 frontage | ✅ | — | 边界 ∩ 道路 |
| 现有 title(s) | ✅ | — | LINZ Titles |
| 形状规整度 | ⏳ | 规整 | 由多边形算 |

## 2. 分区与规划 → `ZoneData`
| 字段 | MVP? | 默认值 | 来源 |
|---|---|---|---|
| Zone | ✅ | — | UnitaryPlanZones(MVP 只做 MHU) |
| 规划版本(Operative / PNPSUD) | ✅ | Operative | UnitaryPlan 各版本服务(**待 planner**) |
| Overlays(洪泛/液化/滑坡/heritage…) | ⏳ | 无命中 | CatchmentsAndHydrology / Geology / ManagementLayers |

## 3. 估值 → `ValuationData`
| 字段 | MVP? | 默认值 | 来源 |
|---|---|---|---|
| CV / LV(最新 LCV/LLV) | ✅ | — | council `Rates24_SnapshotReval`(即用即查) |
| 估值日期 LATESTVALUATIONDATE | ✅ | — | 同上 |
| ⚠️ CV 是 rating 值,非市场卖价 | — | — | 收支里当锚点,卖价另用 $/㎡ |

## 4. 地形 Terrain(地面 = 曲面;平地 = gradient 0)→ `domain/terrain`
| 字段 | MVP? | 默认值 | 来源 |
|---|---|---|---|
| 地面曲面 DTM/DEM | ✅ | — | 本地 1m DEM 栅格(E 盘) |
| 坡度 gradient | ✅ | 0(平地) | DEM 求导 |
| 最低点(排水汇集) | ⏳ | — | DEM |
| 现状物体高度 nDSM | 🔮 | — | DSM − DEM(本地两套栅格) |
| 朝向 aspect / 横坡 cross-fall | 🔮 | 无 / 0 | DEM |

## 5. 基础设施 Infrastructure
| 字段 | MVP? | 来源 |
|---|---|---|
| 最近公共雨水接驳点 + 管底标高 | ⏳ | LiveMaps/stormwater |
| 雨水/污水/供水 容量约束 | 🔮 | UndergroundServices |

## 6. 经济输入 → `domain/economics`(MVP 多为经验默认值,可改)
| 字段 | MVP? | 默认值 |
|---|---|---|
| 建造单价 $/m²、地价、预期售价 | ✅ | 区域经验表(可改);地价可用 LV 锚定 |
| DC 开发贡献费 | ⏳ | 公开费率表 |
| consent / 专业费 / 融资 / GST / contingency | ⏳ | 经验值 |
